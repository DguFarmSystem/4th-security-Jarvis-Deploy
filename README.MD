## Overview
This project is a **web-based SSH access platform** built on top of [Teleport](https://goteleport.com/). It was developed for an Open Source Developer Challenge to provide a **customizable, user-friendly web interface** that overcomes the limitations of Teleport's built-in web UI.
The goal was to decouple the user experience from Teleport's native UI, enabling **localization (Korean support)** and **custom workflows** while leveraging Teleport's robust security features.

## How to run
```
git clone --recurse-submodules https://github.com/DguFarmSystem/4th-security-Jarvis-Deploy.git
cd 4th-security-Jarvis-Deploy
docker build -t jarvis .
docker run -d --name jarvis -p 3000:3000 -p 8080:8080 -p 3025:3025 jarvis
```
You can access the application at http://localhost:3000 and log in with the default credentials


### Core Components
1.  **Teleport Core:** Uses platform-specific Teleport binaries configured with custom `teleport.yaml`.
2.  **Custom Web Server (Frontend & Backend):** Replaces the default Teleport UI to provide a localized and enhanced user experience.
3.  **Dockerized Environment:** A single Dockerfile orchestrates the download of binaries, configuration setup, and simultaneous execution of Teleport, Backend, and Frontend services.

### Decoupled Authentication (CLI Automation)
Since Teleport does not expose public APIs for user registration/login:
- **Backend as Proxy:** The backend acts as an intermediary for authentication.
- **CLI Automation:** When a user logs in via the custom UI, the backend internally executes `tsh login` CLI commands.
- **Certificate Management:** The backend automatically handles the retrieval and storage of user certificates generated by the CLI.

## Challenges
### 1. Lack of Differentiation from Original Teleport
- **Issue:** While we successfully customized the UI/UX and localized the interface, the core functionality remains identical to the original Teleport Web UI. The platform currently lacks unique features that would justify using it over the official client for advanced users.
- **Future Improvement:** To provide real value, we need to implement advanced features not present in the original, such as **AI-driven Log Analysis**, **Anomaly Detection**, or **Custom Audit Dashboards**.

## Lessons Learned
- **Product Delivery:** Learned the complexity of packaging a multi-service application into a single, user-friendly deployable unit.
- **Infrastructure wrapping:** Gained experience in wrapping infrastructure tools (Teleport) with a custom application layer to extend functionality.
- **Deployment Design:** This project provided the foundation for thinking about **deployment strategies** and **container orchestration**.
